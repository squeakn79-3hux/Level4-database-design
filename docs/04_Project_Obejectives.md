# Introduction

The purpose of this document is to define the objectives established during the planning and development of the Inventory Management Database System for Bright Sparks Ltd.

These objectives provided a clear direction throughout the project and ensured that the completed database solution addressed the operational requirements identified within the business scenario.

---

# Primary Objective

The primary objective of this project was to design and implement a relational database capable of replacing manual inventory management processes with a structured, efficient and reliable information management system.

The completed solution was required to support the day-to-day operations of Bright Sparks Ltd while demonstrating industry-standard database design principles.

---

# Project Objectives

To achieve the primary objective, the following goals were established:

- Design a fully normalised relational database.
- Create an Entity Relationship Diagram (ERD) representing the relationships between business entities.
- Develop a MySQL database capable of storing and managing business information.
- Create tables using appropriate primary and foreign key relationships.
- Maintain data integrity through relational database design.
- Develop a graphical user interface using LibreOffice Base.
- Implement forms for efficient data entry.
- Create SQL queries to retrieve meaningful business information.
- Produce reports that assist business decision-making.
- Implement role-based user permissions to improve system security.

---

# Business Objectives

The proposed solution was also expected to support several business objectives.

These included:

- Improve inventory management.
- Increase the accuracy of business records.
- Reduce duplicated information.
- Improve supplier management.
- Improve reporting capabilities.
- Increase staff productivity.
- Support informed purchasing decisions.
- Improve visibility of sales performance.

---

# Technical Objectives

From a technical perspective, the project aimed to demonstrate competency in:

- Relational database design.
- SQL development.
- Database normalisation.
- Entity Relationship Modelling.
- Primary and Foreign Key implementation.
- Query development.
- Report generation.
- User interface development.
- Database security.
- Technical documentation.

---

# Success Criteria

The project would be considered successful if the completed system could:

- Store product information accurately.
- Record supplier information.
- Process customer sales.
- Maintain inventory records.
- Generate meaningful reports.
- Execute SQL queries successfully.
- Demonstrate relationships between all business entities.
- Reduce data redundancy through normalisation.
- Provide a functional and user-friendly interface.

---

# Project Scope

The project focused specifically on the design and implementation of a relational database solution.

The scope included:

- Database planning.
- Database design.
- Database implementation.
- User interface development.
- Forms.
- Queries.
- Reports.
- Testing and evaluation.

The project did not include:

- Web application development.
- Online customer ordering.
- Payment processing.
- Cloud hosting.
- Mobile application development.

These features have been identified as potential future enhancements.

---

# Conclusion

The objectives established for the Inventory Management Database System provided a clear framework for the successful completion of the project.

By meeting both the technical and business objectives, the completed solution demonstrates how a well-designed relational database can improve operational efficiency while supporting informed business decision-making.
