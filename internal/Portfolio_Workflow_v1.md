# Portfolio Workflow v1.0

This is the repeatable workflow for future repositories.

## Phase 1: Foundation

- Create repository
- Clone locally
- Add README
- Add LICENSE
- Add CHANGELOG
- Add .gitignore
- Add folder structure
- Add banner

## Phase 2: Project Intake

Upload one ZIP containing the full project folder exactly as it exists.

Include whatever is available:

- Brief / scenario
- FDD
- ERD
- UNF to 3NF
- SQL
- Screenshots
- Reports
- Forms
- Testing
- Reflection

## Phase 3: Audit

Review the intake ZIP and identify:

- What exists
- What is missing
- What should be public
- What should remain internal

## Phase 4: Build

Create or update:

- README
- docs
- database notes
- diagram notes
- screenshot captions

## Phase 5: Evidence

Add:

- ERD
- screenshots
- SQL export
- reports
- key supporting files

## Phase 6: Polish

Final pass:

- Check links
- Check image paths
- Check spelling
- Check commit history
- Check README readability

## Rule

Keep it clear, short and useful. Evidence beats explanation.
